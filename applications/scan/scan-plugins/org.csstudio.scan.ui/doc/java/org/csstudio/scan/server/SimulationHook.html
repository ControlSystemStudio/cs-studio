    Hook for simulation customization

    The scan configuration file has a <simulation_hook> entry to provide the name of a Jython class that implements a custom simulation hook.

        public boolean handle(ScanCommand command,
             SimulationContext context)
        Called for each command to allow custom simulation

        Implementation should either return false to request default simulation of the command, or perform its own simulation, typically calling SimulationContext#logExecutionStep("text", seconds") with the result, and then return true.

        Parameters:
        command - ScanCommand to simulate
        context - SimulationContext
        Returns:
        true if hook handled the simulation of the command, false if default simulation should be used
