org.csstudio.scan.command

Class ScanScript

  • java.lang.Object
    • org.csstudio.scan.command.ScanScript

  • public class ScanScript
extends java.lang.Object
    Base class for Jython scripts that are executed by a ScriptCommand

    Example: 

      class MyScript(ScanScript):
      def __init__(self, args):
          pass

      def getDeviceNames(self):
         return [ "result1", "result2" ]

      def run(self, context, args):
          [x, y] = context.getData("xpos", "signal")
          context.write("result1", x[0])
          context.write("result2", y[0])

    The jython script that defines the class MyScript must be stored in file named myscript.py, i.e. using the lower case version of the class name.

    All script classes must derive from ScanScript.

      • getDeviceNames

        public java.lang.String[] getDeviceNames()
        Scan scripts that intend to access devices need to list their names. This allows the scan system to connect to the required devices before starting the scan.

        The default implementation returns an empty list, meaning that the script cannot access any 'live' devices. Only data that the scan has logged by the time when the script is invoked will be available.

        Returns:
        Device (alias) names used by the script
