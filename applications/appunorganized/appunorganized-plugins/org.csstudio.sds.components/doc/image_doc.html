Image

This widget displays an image from the given file.

Properties

PropertyName in ImageModelIdentifier Description
FilenamePROP_FILENAMEfilename The file that is to be displayed. If this field is empty, the image will be transparent with nothing drawn on it. If the file could not be opened, the image will display an error message on a filled background. Otherwise, the image in the file will be displayed and possibly stretched and cropped.
StretchPROP_STRETCHstretch If true, the image will be stretched to fit the bounding box.
Crop TopPROP_TOPCROPcrop.top How much of the image, in pixels, is to be cropped on the top.
Crop BottomPROP_BOTTOMCROPcrop.bottom How much of the image, in pixels, is to be cropped on the bottom.
Crop LeftPROP_LEFTCROPcrop.left How much of the image, in pixels, is to be cropped on the left.
Crop RightPROP_RIGHTCROPcrop.right How much of the image, in pixels, is to be cropped on the right.

Implementation

  • the model factory ImageModelFactory

    • Nothing special here, this file is almost the same for all widgets.

  • the model itself, ImageModel

    • This class defines all the properties and provides functions for getting their values (the get* functions).

  • the editpart ImageEditPart

    • This performs initial property synchronisation between the figure and the model (the doCreateFigure function) and registers property listeners. These listeners all just call functions in the figure, which set internal variables.

  • the figure itself, RefreshableImageFigure

    • This is where all the drawing is done. The main class contains functions that outsiders can use to set or get internal variables (the set* and get* functions). The image is drawn using the import org.eclipse.swt.graphics.Image class, which supports BMP, JPG, PNG, GIF, TIFF and ICO file formats.

    • paintFigure paints the image.
