Synoptic Display Studio

Synoptic Display Studio (SDS) ist ein graphisches Operator-Interface, welches die Struktur und den aktuellen Zustand einer (technischen) Anlage abbildet. Die Struktur wird aus Grundelementen, sogenannten Widgets wie 'label', 'meter', 'bargraph' etc. aufgebaut. Um Teile einer Anlage aufzubauen, werden Widgets im Editormodus von SDS auf Displays positioniert. Im Execute-Modus (oder Run-Modus) können die Operateure die Prozesse über die Displays steuern.

First Startup

Erklärung der ersten Schritte mit SDS.

Navigator (red)

Die Navigator-View zeigt die CSS-Projekte im Workspace. Ein CSS-Projekt enthält die Konfigurationsdateien für SDS-Displays und Dateien anderer CSS-Anwendungen.

Editor / Palette (green / blue)

Der Editor zeigt die Displays im Editormodus und stellt 'grid', 'align', 'ruler' etc. bereit. Rechts erscheint die Palette mit allen verfügbaren Widgets. . Es gibt einen drag and drop support der eine Prozessvariable einer anderen CSS-Anwendung automatisch mit einem Widget verbindet.

Widget Properties (yellow)

Für jedes ausgewählte Widget im Editor zeigt die Widget-Eigenschaften-View alle verfügbaren Eigenschaften. Man kann jede Eigenschaft dynamisch verlinken, d.h., dass eine Eigenschaft mit einer Prozessvariablen verbunden wird und sich mit dem Wert der Prozessvariablen ändert.
