Die Anzahl von Aktionen für ein Widget ist nicht eingeschränkt. Im Ausführungsmodus eines Displays zeigt das Cursor-Icon an, dass das darunterliegende Widget Aktionen enthält. Mit einem Rechtsklick werden alle Aktionen des Widgets geöffnet.

Im nächsten Abschnitt gibt es ein Beispiel mit ein paar Aktionen. Es werden nur Prozessvariable mit dem Präfix 'local://' verwendet, damit das Beispiel unabhängig von einem Kontrollsystem bleibt. Weil die folgenden Punkte sich auf Aktionen beziehen, werden sie auch in den Beispiel-Displays erklärt:

Inheritance of Actions: Eine Aktion, die in einem oberen Level definiert ist, ist für die Widgets unterhalb dieses Levels verfügbar.
Group Container: Definieren von ein paar Widgets als Gruppe, um die Vererbung von Aktionen zu demonstrieren.
Alias: Aliase sind wichtig für Aktionen, wenn neue Displays mit Parametern gerufen werden.

Using SDS actions

Das einfachste Beispiel für eine Aktion ist, eine View oder Shell ohne Parameter zu rufen. Rechtsklick auf die Datei 'Calling Display1' → 'Run As Shell'. Wenn man nun einen Rechtsklick auf ein Widget ausführt, sieht man die verfügbaren Aktionen in einem Popup-Menu (s. Figur). Die Widgets erben eine Aktion von der Gruppe. Ausführen der Aktion 'Open as view without aliases' mit Rechtsklick auf die Polyline und Ausführen der Aktion. Das Display 'called display' wird als View in CSS geöffnet.

Um das gleiche Display für verschiedene Anwendungen zu benutzen, kann es mit Parametern gerufen werden. Rechtsklick auf die Polyline in der laufenden Shell 'Calling Display 1' und Auswählen der Aktion 'Open as Shell with aliases'.
Öffnen 'calling display 2': Rechtsklick auf die Datei 'Calling display 2' → 'Run as Shell'. Rechtsklick auf die polyline in der laufenden Shell 'Calling Display 2' und Auswählen der Aktion 'Open as Shell with aliases'.
Das 'called display' wird in zwei Shells mit unterschiedlichen Parametern ausgeführt. Wenn man jetzt die Aktion 'Send a value' von polyline, ellipse oder group aus ausführt, sieht man den Wert im Label des Calling Displays. Der Label des 'called displays' zeigt den Wert, der im 'calling display' gesetzt ist, weil das 'called display' mit dem entsprechenden Kanal parametrisiert ist. Die Aktion 'Call As Shell' setzt über die Eigenschaft 'alias' einen Aliasnamen für Widgets, um Parameter zu empfangen. Um einen Wert von 'Send action' auf einen Kanal zu schreiben, muss man die Eigenschaft 'Action data' dynamisieren, wie im Beispiel gezeigt wird. Der Kanal, der für den 'output channel' definiert ist, wird von 'Send action' benutzt.
