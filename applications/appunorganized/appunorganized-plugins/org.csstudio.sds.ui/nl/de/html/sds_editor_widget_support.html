Drag and drop support for widgets

Wenn man die Widget-Palette zum Hinzufügen neuer Widgets zu einem Display benutzt, benötigt man auch Editor-Eigenschaften, um Widgets mit dem Control-System zu verbinden. Der 'Drag and Drop'-Support setzt diese Eigenschaften automatisch. 'Drag' ein Objekt, welches mit einer Prozess-Variablen verbunden ist, aus irgendeiner Control-System-Anwendung in das SDS-Editor-Fenster. Der Screenshot zeigt ein Beispiel mit der Anwendung 'PV-Table'.




Nach dem 'drop'pen des Objekts erscheint ein Fenster, in dem man den Typ des Control-Systems auswählen kann, mit dem die Prozess-Variable verbunden werden soll.


Das nächste Fenster zeig eine Auswahlliste aller Widgets, aus dem man das passsende Widget auswählen kann.


Im letzten Window ist ein Alias für eine Prozess-Variable gesetzt. Der Alias kann für die Eigenschaften eines Widgets benutzt werden; es erspart das mehrfache Tippen des ganzen Namens der Prozess-Variablen.




Das neue Widget ist jetzt mit allen Settings für die Prozess-Variable initialisiert. Ein einfacher 'Label' ist nur mit dem Wert der Variablen verbunden, während für komplexere Widgets, wie z.B. 'bar Graph' auch Alarm Level, High / Low Displays etc. definiert sind.

Preferences

Der Initialisator für Widgets wird in den 'Preference Pages' gesetzt:

Select Menu CSS → Preferences...
Open Preference Page CSS-Applications → Display → Default Shema
Mit der Option 'manual' wird nur der Alias für Prozess-Variable gesetzt.
Die Option 'EPICS' setzt den Initialisator so, dass alle normalen Eigenschaften eines Epics-Record für Prozess-Variable gesetzt werden.
